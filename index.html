Fire Alarm Detection on Cisco Webex Video Devices

Cisco Webex video devices can detect T3 fire alarms. Having dectected the alarm, the device can perform multiple actions including displaying alerts and exit maps, sending webhooks or API calls to faclities or building security systems.

In order for the device to do something when it hears the T3 alarm, it needs a custom macro installed. This macro runs in the RoomOS Macro Framework and contains the commands the codec will execute upon detecting an alarm

See my example macro here - https://github.com/tyosgood/T3alarm/blob/main/FireAlarmAlert.js

T3 is the standard alarm pattern for fire alarms in the United States and many other countries. In the US, the use of T3 alarm coding was adopted into the fire code by the NFPA in 1996. As a result, all building built in the US since 1996 are required to use alarms that emit the T3 pattern.

As per NFPA 72, 18.4.2 (2010 Edition)Temporal Code 3 is the standard audible notification in a modern system. It consists of a repeated three-pulse cycle (0.5 s on, 0.5 s off, 0.5 s on, 0.5 s off, 0.5 s on, 1.5 s off). Voice Evacuation is the second most common audible in a modern system. Legacy systems, typically found in older schools and buildings, have used continuous tones alongside other audible schemas. Source: https://en.wikipedia.org/wiki/Fire_alarm_system

See also - https://www.nfpa.org/News-and-Research/Publications-and-media/Blogs-Landing-Page/NFPA-Today/Blog-Posts/2021/05/21/A-Guide-to-Fire-Alarm-Basics-Notification?icid=W483

The device can be programmed to do almost anything when it dectects an alarm. For example, you can display an image or webpage, send notifications to external systems or users via API or webhook, place a call to a spefic location. You can do anything the device can do via the xAPI and the macro framework.
